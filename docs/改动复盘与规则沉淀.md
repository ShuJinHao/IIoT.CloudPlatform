## 2026-07-02 AI 高频查询 API 收口

- 改动范围：Cloud AI Read 权限、Human 权限定义、AI Read Controller、ProductionService 查询用例、Dapper 读服务、服务契约、ServiceLayer/ProductionService/E2E 测试和 Cloud 规则文档。未修改 `IIoT.EdgeClient`、`AICopilot`，未执行部署。
- 改动原因：AI 项目后续只把 Text-to-SQL 作为低频探索兜底；设备日志、设备产能和生产数据这类高频查询必须由 Cloud 提供正式只读 API，避免 AI 端直接查生产库、绕过权限、审计、限流和设备授权范围。
- 影响面：新增 `AiRead.ProductionRecord` 权限点；新增 `GET /api/v1/ai/read/production-records` 通用生产数据查询；新增 `GET /api/v1/ai/read/capacity/hourly` 小时产能查询；设备日志 AI 查询支持 `preset` 和 `minLevel`；移除 AI 专用 `/api/v1/ai/read/pass-stations/{typeKey}` 语义入口。Human 过站接口和 Edge 上传接口不变。
- 关键实现：通用生产数据查询读取 `pass_station_records` 公共列并按 `pass-station-types.json` 返回 schema 化字段，支持 `typeKey`、`processId`、`deviceId`、`barcode`、`result`、显式时间范围或 `preset`；`fieldMode=list/full` 控制字段集合；不向 AI 返回 raw `payload_jsonb` 字符串。所有新增 AI Query 继续走现有 `IAiReadQuery<T>` + MediatR pipeline，复用鉴权、审计、限流和设备授权 scope。
- 规则提炼：形成长期规则，已写入 `docs/云端规则.md` 和 `docs/过站工序扩展规则.md`：AI 高频业务读取必须通过 Cloud `/api/v1/ai/read/*` 只读 API；通用生产数据只使用 `/api/v1/ai/read/production-records`；Text-to-SQL 只作为低频探索兜底；新工序进入 `pass_station_records + pass-station-types.json` 后默认具备 AI 通用读取能力。

## 2026-07-01 Edge 发布 API key 稳定通道

- 改动范围：Cloud 后端新增 Edge Release API key 持久化、机器发布 JWT 签发接口、Human 管理 API、JWT claim、权限管道、Gateway machine surface、OpenAPI route surface 和 EF migration；同步更新 Cloud 规则和复盘。未修改 Cloud 前端、Edge 安装包产物、Cloud/AICopilot 部署脚本或现有发布包校验/保留策略。
- 改动原因：Edge 本机发布凭据使用 Human refresh token 时会受 Human 活跃会话上限影响，同账号多端登录会导致 Keychain 中 refresh token 被服务端撤销，部署 preflight 反复 401。用户要求形成稳定发布渠道，不能每次部署都要求重新提供账号密码。
- 影响面：新增 `edge_release_api_keys` 表；新增 `POST /api/v1/machine/edge-release/token` 用 API key 换短期发布 JWT；Gateway 正式放行 `/api/v1/machine/{**catch-all}` 并标记 `X-IIoT-Route-Surface: machine`；新增 `/api/v1/human/client-release-api-keys` 创建/列表/吊销接口。现有 `/api/v1/human/client-releases/*` 上传、catalog、审计和保留策略继续复用，不新增绕过发布审计的旁路。
- 关键实现：API key 明文只创建时返回一次，数据库只保存 SHA256 hash；机器 JWT 使用 `actor_type=edge-release-publisher`，只携带 `ClientRelease.Read` / `ClientRelease.Publish`；`AuthorizationBehavior` 路线 A 扩展为机器 actor 直接校验 permission claims，不改现有发布命令 `IHumanCommand` 类型，也不改 `RequestKindGuardBehavior` 分类规则；机器 actor 不能执行 `ClientRelease.Manage` 或未声明权限点的 human 请求；API key 创建/吊销写 `ClientRelease.ApiKey.Create/Revoke` 审计，发布上传继续复用 `ClientRelease.Publish` 审计，机器发布审计中 `ActorUserId` 可为空，`ActorEmployeeNo` 显示 `edge-publisher:<keyName>`。
- 规则提炼：形成长期规则，已写入 `docs/云端规则.md` 和工作区 `docs/上传部署总览.md` / `docs/总规则.md`：Edge 日常发布使用 Edge Release API key，不依赖 Human refresh token；机器发布身份权限必须限定为发布读取和上传，不得获得管理/删除能力。
