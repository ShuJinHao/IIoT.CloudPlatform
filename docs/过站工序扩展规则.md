# 过站工序扩展规则

本文档约束 `IIoT.CloudPlatform` 的过站数据扩展方式。当前云端已经统一为“公共检索列 + `payload_jsonb` 扩展字段 + 配置驱动 schema”的模型。

## 固定规则

- 新增普通过站工序时，默认只改 `src/hosts/IIoT.HttpApi/config/pass-station-types.json`。
- 不再为每个工序新增独立 DTO、Event、Mapper、SQL、Consumer、Vue 页面或 RabbitMQ endpoint。
- 统一上传接口为 `POST /api/v1/edge/pass-stations/{typeKey}/batch`。
- 统一查询接口为 `GET /api/v1/human/pass-stations/{typeKey}` 和 `GET /api/v1/human/pass-stations/{typeKey}/{id}`。
- 前端过站页面必须从云端 `GET /api/v1/human/pass-stations/types` 读取 schema 动态渲染。
- 工序扩展字段必须进入 `payload_jsonb`，公共检索锚点保留为普通列。

## 公共列

`pass_station_records` 固定保留以下普通列：

- `id`
- `type_key`
- `device_id`
- `barcode`
- `cell_result`
- `completed_time`
- `received_at`
- `deduplication_key`
- `payload_jsonb`

只有这些字段可以直接参与通用高频过滤和排序。新增工序字段如果需要高频过滤、排序或统计，必须单独评估是否提升为普通列，或新增 JSONB 表达式索引。

## 配置项

每个工序配置必须包含：

- `typeKey`：工序类型键，必须唯一，推荐与工序编码的小写形式一致。
- `displayName`：中文显示名。
- `description`：前端页面说明。
- `supportedModes`：支持的查询模式。
- `fields`：工序扩展字段定义。
- `listColumns`：列表展示列。
- `detailSections`：详情页分组展示字段。

字段支持类型：

- `string`
- `number`
- `integer`
- `boolean`
- `datetime`
- `enum`

字段支持规则：

- `required`
- `maxLength`
- `min`
- `max`
- `unit`
- `precision`
- `options`

云端启动时会校验配置。`typeKey` 重复、字段 key 重复、字段 key 不安全、列表或详情引用不存在字段，都会导致服务启动失败。

## 新增工序步骤

以新增“匀浆”为例：

1. 在 `src/hosts/IIoT.HttpApi/config/pass-station-types.json` 增加一个 `types` 配置项。
2. 设置 `typeKey`，例如 `homogenization`。
3. 在 `fields` 中定义匀浆专属字段，例如 `slurryBatchNo`、`viscosity`、`solidContent`。
4. 在 `listColumns` 中选择列表展示字段。
5. 在 `detailSections` 中安排详情页分组。
6. 运行后端构建和前端构建，确认 schema 校验、上传校验和页面渲染通过。

除非新增公共检索字段、全新业务链路或新的持久化策略，否则不得新增 per-process 代码文件。

## 允许新增代码的例外

只有以下情况可以突破“只改配置”的规则：

- 新字段需要成为系统级公共检索锚点。
- 需要按 JSON 内字段做高频过滤、排序或聚合统计。
- 需要全新的业务链路，不再属于普通过站追溯。
- 需要独立权限、审计或状态流转规则。

出现这些情况时，必须先说明影响范围，再单独开小 PR，不得混入普通工序扩展。

## 验证要求

新增或调整工序配置后，至少执行：

```powershell
dotnet build IIoT.CloudPlatform.slnx -c Release
dotnet test src/tests/IIoT.ServiceLayer.Tests/IIoT.ServiceLayer.Tests.csproj -c Release
cd src/ui/iiot-web
npm run build
```

如果改动涉及数据库表、上传链路、Outbox 或部署配置，还需要运行对应 E2E 和 compose 校验。
