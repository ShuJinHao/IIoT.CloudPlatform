## 测试执行口径

- 测试分类只有 `Architecture`、`Security`、`Business`、`DeploymentContract`、`Quality`、`CrossProject`。
- `Architecture` 与 `Security` 是不可被普通业务改动弱化的门禁；`Business` 可随业务同批新增、修改或删除。
- 默认开发入口是 `scripts/tests/Select-CloudCiTests.ps1 -Mode Default`，随后由 `Invoke-CloudCiSelectedTests.ps1` 执行本次受影响选择并生成 `current-discovery.json`。
- 普通部署只允许工作区 `deploy/Deploy-Changed.ps1 -Targets Cloud` 使用 `Deployment` 选择并复用同 SHA 证据；部署任务不改源码、测试或文档。
- `Quality`、`Full`、`CrossProject`、完整 E2E、coverage、mutation、duplication 和三端对齐仅在用户当前轮明确授权时运行。
- 影响无法归属时停止并报告具体文件；不得退化为全量。
- 仓库不保存固定 case 数 inventory、业务测试数量下限或 coverage case baseline。

## 当前验证示例

```powershell
pwsh scripts/tests/Select-CloudCiTests.ps1 `
  -Mode Default `
  -BaseRef <真实祖先 SHA> `
  -HeadRef HEAD `
  -OutputPath artifacts/ci-selection.json

pwsh scripts/tests/Invoke-CloudCiSelectedTests.ps1 `
  -SelectionPath artifacts/ci-selection.json `
  -Configuration Release
```

只修改 selector 时运行 `scripts/tests/Test-CloudCiTestSelection.ps1`。跨仓验证必须显式选择 `CrossProject` 并设置精确 `AICOPILOT_REPOSITORY_ROOT`；不得由普通 push、部署或 nightly 自动触发。
