# 云端安全部署契约

本文档约束 `IIoT.CloudPlatform` 当前生产部署安全边界。总规则见 `../../docs/总规则.md`，云端项目规则见 `云端规则.md`。

## 1. 当前传输基线

- 当前生产部署必须使用内网受控 HTTP。
- 当前批次不启用 HTTPS/TLS/443/HSTS，不做 HTTP 到 HTTPS 跳转。
- 证书采购、证书签发、证书轮换和 HTTPS 改造不进入当前治理计划的阻断项。
- 未经用户明确批准证书预算、证书来源、变更窗口和允许修改目录，不得新增 nginx 443 listener、TLS 证书挂载、HSTS、HTTPS redirection 或应用层 `UseHttpsRedirection`。

## 2. HTTP 使用边界

- HTTP 只允许用于受控内网、VPN 或等价专线网络。
- Cloud 不得直接暴露到公网 HTTP。
- 一旦需要公网访问、外部客户访问、跨互联网访问或跨不可信网络访问，必须先暂停发布并重新评审传输安全方案。
- `PUBLIC_BASE_URL` 和 Edge 发布下载地址可以是 HTTP，但必须指向受控内网地址或内网域名。
- OIDC issuer、AICopilot redirect URI 和 post-logout redirect URI 的 HTTP 例外更严格：只有 `ALLOW_INTRANET_HTTP_OIDC=true` 且 host 是 loopback 或 RFC1918 私网 IPv4 时才允许；公网 IP、普通域名、模板域名和未知 host 的 HTTP OIDC 必须拒绝。`.env.example` 中的 `cloud.internal.example` / `aicopilot.internal.example` 只是模板占位，不能作为 HTTP OIDC 生产值上线。
- `deploy/README.md` 的生产 HTTP OIDC 示例必须使用 `<cloud-private-ip>` / `<aicopilot-private-ip>` 这类私网 IPv4 占位，避免把模板域名误当成可部署值。

## 3. 必须保留的补偿控制

- Bootstrap secret 恒必需；不得再保留 `RequireSecret=false` 或等价旁路。
- JWT secret、数据库密码、RabbitMQ 密码、Seq 密码、OIDC 证书密码、Edge Release API key 必须由部署 secret 管理，不得提交仓库。
- `.env.example` 不得包含真实生产 secret；示例值只能是不可直接上线的占位值。
- 部署前脚本必须 fail-fast 检查弱 secret、空 secret、bootstrap secret 关闭、Docker Hub 生产依赖、远端候选不可解析和 compose 配置错误。日常入口从配置远端 tip 建隔离 worktree，因此本地业务工作区可以脏，但未提交/未推送内容绝不能进入镜像。
- Human、Edge、Machine、AI Read 四类入口必须分离；AI 只读入口不得暴露写 API。
- self-hosted runner 不得以 root 运行；日常生产发布不得依赖 runner 持有完整生产密钥。
- SSH 发布应使用专用部署用户和受限 sudo；`root@host` 只能作为已批准的应急路径。
- 当前 `Cloud` 与 `AICopilot` 共用同一台生产宿主机时，必须区分“共享宿主机 / 共享发布人”与“各自独立部署根权限状态”；Cloud 根的 root-owned 漂移不得被误写成整机通用故障，也不得因为 AICopilot 同机正常就跳过 Cloud 根修复。
- root 应急路径如果写入了 `certs/`、`releases/*` 或其它持久化部署状态，关闭任务前必须恢复 owner/mode，并重新验证标准 non-root preflight。
- Cloud 日常应用发布不得同步 compose、nginx、服务器 scripts 或 Runner；它只发送 digest-bound 不可变镜像请求。部署基础设施变更必须拆成显式维护任务，继续使用白名单 staging/SHA-256 旧事务链并排除 `.env`、`certs/`、`releases/`、`backups/`，维护完成后独立升级 Runner 和执行 Doctor。
- Cloud release 与 post-release cleanup 必须使用可识别 live/initializing/stale 状态的 managed lock；活锁 fail-fast，stale lock 只有在 PID/进程启动标识证明原进程失效后才能原子认领并清理。不得再次用无状态目录锁静默等待 15 分钟。
- release transaction 必须在 support 变更前原子写 marker并扫描 marker/lock/staging/recovery；无 promotion proof 才允许恢复，已证明 promotion 后任何 cleanup 路径都不得恢复旧 support。local stale/malformed lock precheck 只读，阻断证据保留旧 invocation 和所有恢复路径。
- installer/release child 必须从 `env -i` 白名单环境在独立进程组启动，存活判断必须检查 PGID 而不是仅检查组长 PID。组长已退出但子孙仍存活时，仍必须有界 TERM/KILL，确认整组消失才能恢复/释锁。dotenv 错误仅允许 file/line/category，不得输出敏感 key/value/token。
- 操作员 `.env` 必须与 release-owned 镜像状态分离；deploy/rollback 不得替换 `.env`，镜像只能经 `staged-images.env` 原子晋升到 `current-images.env`。deploy、rollback 和 `update-deploy-env.sh` 必须共用严格配置锁；锁存在即返回 75，不得 stale 自动删除。直接外部写 `.env` 无法受互斥保证，属于不受支持操作；如果在最终晋升前检测到，必须保留操作员值并 abort。
- 标准日常入口必须实时输出阶段。中止或远端失败后必须先核对 `routine-current.env`、容器镜像和锁；已生成请求时优先用 `-ResumeInvocation` 重发，不得重新 build。cleanup/GC/深度巡检移出同步热路径，不能把健康应用因清理告警误报为发布失败。
- nginx 必须保留安全头、限流、上传体积限制、目录 listing 禁止和 `/internal/healthz` 本机访问限制。
- OIDC token 发行验收必须走真实 authorization-code + PKCE 流程；post-deploy 脚本只能在 `POST_DEPLOY_VERIFY_OIDC_TOKEN=1` 且操作者通过 0600 类私有文件提供真实一次性 code/verifier 时调用 `/connect/token`，不得伪造密码流或 client_credentials，也不得把 code/verifier 放进日志、文档、shell history、进程环境或 curl 进程参数。
- 日志不得输出 token、secret、连接串、密码、OIDC code、refresh token 或 Edge binding payload 中的敏感字段。

## 4. 当前不做的事项

- 不采购或配置公网 CA 证书。
- 不引入自签证书作为默认生产路径。
- 不增加 443 端口和 TLS 证书挂载。
- 不启用 HSTS。
- 不启用应用层 HTTPS redirection。
- 不把 HTTP-only 本身作为 CI 失败项。

## 5. 允许后续升级的条件

只有同时满足以下条件时，才允许重新评审 HTTPS/TLS：

- 用户明确批准证书预算或提供可用证书来源。
- 明确证书存放、轮换、备份和权限方案。
- 明确是否影响 EdgeClient、AICopilot、OIDC issuer、下载地址和现场脚本。
- 明确回滚路径和停机窗口。
- 同步更新 `云端规则.md`、本契约、部署文档、nginx/compose、preflight 和 `DeploymentSourceGuardTests`。

## 6. 验收命令

```bash
rg -n "云端部署传输红线|内网受控 HTTP|不得新增 nginx 443|不启用 HTTPS/TLS" IIoT.CloudPlatform/docs
rg -n "listen 443|Strict-Transport-Security|UseHttpsRedirection|GATEWAY_HTTPS_PORT" IIoT.CloudPlatform/deploy IIoT.CloudPlatform/src -g '*.conf' -g '*.yml' -g '*.cs' -g '*.md'
dotnet test src/tests/IIoT.CloudPlatform.DeploymentTests/IIoT.CloudPlatform.DeploymentTests.csproj -c Release --no-build --no-restore --disable-build-servers
```
