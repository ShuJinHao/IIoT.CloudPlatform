# 设备客户端状态投影契约

本文档约束 Cloud 侧客户端安装状态、运行心跳和设备客户端状态查询。

## 1. 官方投影

- `DeviceClientState` 是客户端状态官方投影，按 `DeviceId + ClientCode` 唯一。
- 版本快照、运行心跳和状态投影不是业务聚合根，不得通过通用 `IRepository` / `IReadRepository` 暴露。
- 客户端状态读写必须通过 `IDeviceClientStateStore`。
- 页面、AI Read 和后续状态类查询不得临时拼接 `DeviceClientVersionSnapshot` 和 `EdgeDeviceRuntimeHeartbeat` 重新解释状态。

## 2. 写入链路

- 版本上报必须校验 `DeviceId + ClientCode` 身份链。
- 版本上报写入或替换 `DeviceClientVersionSnapshot` 后，必须同步 `DeviceClientState.ApplyVersionReport`。
- 运行心跳必须校验 `DeviceId + ClientCode` 身份链。
- 运行心跳写入或替换 `EdgeDeviceRuntimeHeartbeat` 后，必须同步 `DeviceClientState.ApplyRuntimeHeartbeat`。
- 缺少状态投影时，写入链路必须创建 `DeviceClientState`，不得只保留原始事实表。

## 3. 读取链路

- Human 设备客户端状态清单必须以 `DeviceClientState` 为权威来源。
- Human 清单可以读取版本快照用于插件明细，但安装状态、软件状态、当前版本、上报时间和心跳时间必须优先来自 `DeviceClientState`。
- AI Read 的设备客户端状态只能通过 `GetAiReadDeviceClientStatesQuery` 读取 `DeviceClientState` 投影，并继续执行 AI 授权范围过滤。
- 运行心跳不得冒充 Cloud 上传状态、MES 上传状态或安装状态；没有真实来源时字段保持空或明确问题说明。

## 4. 禁止项

- 禁止新增 `IRepository<DeviceClientState>`、`IReadRepository<DeviceClientState>` 或面向状态投影的通用 Specification。
- 禁止在 Controller、页面查询或 AI Read 中直接拼接版本快照和运行心跳来推断状态。
- 禁止让运行心跳超过 24 小时代替“客户端未运行”判断。
