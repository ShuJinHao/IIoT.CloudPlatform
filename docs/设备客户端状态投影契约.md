# 设备客户端状态投影契约

本文档约束 Cloud 侧客户端安装状态、运行心跳和设备客户端状态查询。

## 1. 官方投影

- `DeviceClientState` 是客户端状态官方投影，按 `DeviceId + ClientCode` 唯一。
- 版本快照、运行心跳和状态投影不是业务聚合根，不得通过通用 `IRepository` / `IReadRepository` 暴露。
- 客户端状态读写必须通过 `IDeviceClientStateStore`。
- 页面、AI Read 和后续状态类查询不得临时拼接 `DeviceClientVersionSnapshot` 和 `EdgeDeviceRuntimeHeartbeat` 重新解释状态。

## 2. 写入链路

- 版本上报必须校验 `DeviceId + ClientCode` 身份链。
- 版本上报写入或替换 `DeviceClientVersionSnapshot` 后，必须同步 `DeviceClientState.ApplyVersionReport`。
- 运行心跳必须校验 `DeviceId + ClientCode` 身份链。
- 运行心跳写入或替换 `EdgeDeviceRuntimeHeartbeat` 后，必须同步 `DeviceClientState.ApplyRuntimeHeartbeat`。
- 缺少状态投影时，写入链路必须创建 `DeviceClientState`，不得只保留原始事实表。

## 3. 读取链路

- Human 设备客户端状态清单必须以 `DeviceClientState` 为权威来源。
- Human 清单可以读取版本快照用于插件明细，但安装状态、软件状态、当前版本、上报时间和心跳时间必须优先来自 `DeviceClientState`。
- Human 统一“设备运行与版本”主查询使用独立窄权限 `DeviceClientOverview.Read`，以当前用户可访问的 `Device` 为主集，按当前 `DeviceId + Device.Code` 左连 `DeviceClientState`；无投影的授权设备仍返回 `MissingRuntimeHeartbeat` 行。
- 统一主查询只返回设备 ID/名称、IP、`softwareStatus`、当前版本和异常摘要。授权设备过滤、keyword、count、排序、`Skip/Take` 必须在数据库执行，应用层只允许按当前页设备 ID 批量读取状态投影。
- PLC 详情必须经 `EdgeHost.Read` 查询现有 PLC runtime state；版本、插件和升级详情必须经 `ClientRelease.Read` 查询。主查询权限不得隐式获得或预取这两类详情，无权限时不得返回占位详情或伪造空事实。
- AI Read 的设备客端状态只能通过 `GetAiReadDeviceClientStatesQuery` 读取；列表必须以 delegated scope 内的当前 `Device` 为主集，再按 `DeviceId + 当前 Device.Code` 左连一条 `DeviceClientState`。
- AI Read 状态查询支持 `deviceId`、`deviceCode`、`processId`、`keyword`、`maxRows`，所有条件与 delegated device scope 必须在同一数据库查询中完成 AND 过滤、count、稳定排序和分页。`RowCount` 是当前页设备数，`Truncated` 只由过滤后总设备数是否大于当前页行数决定。
- 授权设备即使尚无投影也必须返回一行：`clientCode` 使用当前 `Device.Code`，`softwareStatus=MissingRuntimeHeartbeat`，投影专属字段和 `updatedAtUtc` 保持 `null`，不得伪造上报时间。
- 运行心跳不得冒充 Cloud 上传状态、MES 上传状态或安装状态；没有真实来源时字段保持空或明确问题说明。

## 4. 软件状态语义

- `runtimeStatus` 是 Edge 最新运行心跳中的原始状态；`lastRuntimeHeartbeatAtUtc` 是唯一新鲜度时间。版本上报时间、投影 `UpdatedAtUtc` 和版本号都不得改变运行新鲜度。
- `softwareStatus` 只能由共享纯判定器根据 `DeviceClientState? + 显式 utcNow` 生成：`MissingRuntimeHeartbeat`、`RuntimeHeartbeatStale`、`Starting`、`Running`、`Stopped`、`Unknown`。
- 仅当 `utcNow - lastRuntimeHeartbeatAtUtc > 24h` 时为 `RuntimeHeartbeatStale`；恰好 24 小时仍是新鲜。`Stopping` 和 `Stopped` 统一投影为 `Stopped`，其它未识别原始值投影为 `Unknown`。
- Human 上位机列表/详情与 Human 客户端版本清单必须复用同一判定器，并在每次 handler 内只捕获一次 `utcNow`。Human 现有 `Issue` 文案保持“客户端尚未上报运行心跳。”/“超过 24 小时未收到运行心跳。”；AI DTO 不暴露 `issue`。

## 5. 禁止项

- 禁止新增 `IRepository<DeviceClientState>`、`IReadRepository<DeviceClientState>` 或面向状态投影的通用 Specification。
- 禁止在 Controller、页面查询或 AI Read 中直接拼接版本快照和运行心跳来推断状态。
- 禁止用版本上报、版本号或投影更新时间替代运行心跳判断客户端软件状态。
- 禁止用 `DeviceClientOverview.Read` 返回 PLC runtime state、插件清单、升级详情或发布管理数据，也禁止为了详情字段把全量授权设备/状态拉入内存后分页。
